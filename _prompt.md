comfyuiのprompt wordを記録するgoogle chrome 拡張機能を作る。拡張機能のICONをクリックするとポップアップする形。

prompt wordのグルーブ分けが重要、例えばキャラクター、背景、光のグループがあり、キャラクターの中にはさらに上半身、装飾品、下半身などのグループがあり、グループ名もprompt wordも自由で記録できる。

prompt wordを選定すると、総括の欄に全prompt wordを纏めた(重複wordは排除)最終promptを出力。

wordが多く積むとグループの列がすごく長くて、選択することが困難になるため、グループに折り畳み機能を追加して欲しい、そのグループの中に選択されたwordがある場合はグループに何かしらのヒントがあるといい、あとはwordは後から追加するので、読みやすさのため、wordの順とグループの順は並び直しができるように、ドラッグ&ドロップを実装して。ドラッグ&ドロップは何かしらのスムーズな動画エフェクトがあるといい。

wordはグループ内のみ動くが可能。グループは順以外、AグループからBグループの中に移すも可能。

wordにクリックする時、シングルクリックは選択、ダブルクリックは編集。
グループ名にクリックする時、シングルクリックは階層の展開と閉め、ダブルクリックは編集。

wordには注釈の機能が必要、こうすると意味が分からないwordもすぐに理解するようにできます。

wordと注釈から検索する検索欄も必要。

最後にjsonファイルとして書き出しと読み込みの機能も必要。書き出しと読み込みのUIはICONだけでいい、文字は不要。

画面のレイアウトについて、左がwordの画面, 右の上半部は総括欄の画面、下半部は選ばれたword、このwordをクリックすると選択がキャンセルする仕組み。左の範囲は多め、右は少なめ、画面の調整は自然界のルールを従って任せる。

### 機能：
- インデクスにホバーまたは針の指針を指すとグループ名をインデクスの近くにポップアップで表示。
- インデクスまたは指針をクリックすると時計形状のロードマップは消え、目標のグループにスクロールしていきます。
- スクロールの目標グループが閉じたは自動で展開。

時計型ロードマップのインデックスを立体的階層構造に設計変更、一層（最外層）はルードグループ(Aと呼ぶ)とAの子グループ(Bと呼ぶ)、二層（内側の層）はBの子グループ(Cと呼ぶ)とCの子グループ、このような形で階層を重なっていく。BとCの間は一本の線が繋がって視覚的に階層の間の関係性を示す。時計型の同一階層のインデックス親と子は視覚で区別できるように。
針の長さも階層に合わせて長さ自動変更するように。針は一本だけですので、ちゃんと針が全部のインデックスに指すことができるように各層のインデックス配置位置を調整、内の層は外の層のインデックスに被せると針から指すことができないので。

## プリセット機能の調整と変更を行う
### プリセットの保存機能の調整。
- プリセットの保存ボタンをクリックすると、プリセット保存の詳細なフォームがポップアップで現れます。
- フォーム内は以下の情報を記入、
```ts
{
  name: string, baseModel: string, baseModelKind: string, loras?: {model: string, strength: number}[], controlNets?: {model: string, strength: number}[], metadata: {
    steps: number, cfg: number, sampler: string, scheduler: string, width: number, height: number
  },
  image: string,
  description?: string
}
```
imageプロパティについて、これはユーザーから送った画像を圧縮して、Base64文字列として保存するもの、画像圧縮の関数は既に/src/image.tsの中に書いておりましたので、そちら関数を使用、今回圧縮の最大幅は560pxに調整。画像を圧縮する前に、exiftからwidthとheightを自動検出し、フォームに自動記入。
- あとは復元に使われたワードのid, text, strengthを保存、これで一つのプリセットは保存完了になります。ここのテキストは還元時もしidがなかった時、textの内容が変わった時、ユーザーに情報を知らせるためです、textの内容自体は還元しない。

### プリセットの還元機能
- 元の「保存済みプリセット」ボタンの関連機能は全部廃止、プリセット一覧ボタンで置き換える。
- プリセット一覧ボタンをクリックするとプリセット一覧のブロック（サイズは全画面）が右から左へスライドしてきます。
- 中にはプリセットは正六角形のブロックとして並び、プリセットの背景はimageプロパティの画像、プリセットの真上はname, 左上は還元ボタン、右上はbaseModelKind、真下は編集、更新、削除ボタン。プリントにhoverすると還元ボタン、編集ボタン、更新ボタン、削除ボタンは現れます、普段は見えないようにする。
  - ** 還元ボタン ** 還元ボタンを押すと確認のダイアログをポップアップし、ユーザーの確認を行ってから還元を行う。還元時はidとtextを確認し、失ったidまたは変更されたtextがあった場合はユーザーに知らせ。
  - ** 編集ボタン ** 編集可能なプロパティはname, baseModel, baseModelKind, loras, controlNets, metadataとなります。
  - ** 更新ボタン ** 更新するのは現在のワード情報になります。更新ボタンをクリックすると、前の情報と比較し、追加、削除、強度変化、テキスト変化の差分情報をポップアップして、ユーザーがokを押したら、プリセットの更新を行う。
  - ** 削除ボタン ** 削除ボタンを押すと確認のダイアログをポップアップし、ユーザーの確認を行ってから削除を行う。
- プリセットの順はドラッグ&ドロップで上下、左右位置調整可能。
- プリセットをクリックすると、立体なcard(translate3d)がポップで現れます。このcardは3Dマウスホバーエフェクトがあり、マウスの動きに追従して豪華なエフェックスが演出、この効果はできるだけ派手に、高級感とゴージャス感を演出して。
  - cardの高さは560px, cardの背景にはimageプロパティの画像。
  - cardを左クリックまたはcardにある回転ボタンをクリックすると、cardはY軸で回転、背面を表す。
  - cardの背面にはbaseModel, baseModelKind, loras, controlNets, metadata, descriptionの情報を記載。
