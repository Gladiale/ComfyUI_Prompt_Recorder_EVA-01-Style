I

概要 / Overview

ComfyUI で使用するプロンプトワードを、自由に階層化されたグループへ記録・選定し、重複を排除した最終プロンプトを生成するブラウザ拡張機能。アイコンクリックでポップアップ起動。

目的

プロンプトワードを意味的にグループ化（キャラクター / 背景 / 光 等、更にその中に上半身・装飾品・下半身等のサブグループ）して蓄積。必要なワードだけを選ぶことで、最終プロンプトを即時に組み立てる。

形態

Google Chrome 拡張機能（Manifest V3）。拡張機能アイコンをクリックするとポップアップ（popup.html）が開き、その中に全 UI を展開する。状態は chrome.storage.local に永続化。

タイポグラフィ

見出しに宗教典籍を思わせる Cinzel / Cinzel Decorative（ローマ碑文体）。本文に繊細で古典的な EB Garamond。データ・コード類には JetBrains Mono を採用。

神秘的さ＝「静寂・光・碑文」。文字間を広めに、グロー（発光）で聖性を、紫の深淵で畏敬を表現。

ビジュアル演出

  • 背景：深淵の紫黒に、微かな格子と光のハロー（aurora halation）。
  • 選択状態：ワードが蛍光緑で縁取られ、inner-glow + 発光で「起動」を表現。
  • 境界線：細い青紫のライン、要所に緑の強調線（初号機の装甲継ぎ目を模倣）。
  • ドラッグ中：対象が半透過し、緑の残像Trail（Motion の spring）を伴う。
III

画面レイアウト / Layout

左右非対称。自然界の比率（黄金比 ≈ 1.618）に従い、左に広く、右に控えめに。

区分

  • 左領域（広）：ワード画面。全グループ／ワードの閲覧・編集・ドラッグ。
  • 右上（半）：総括欄。選択ワードの重複排除済み最終プロンプトを出力。
  • 右下（半）：選択済みワード一覧。各ワードをクリックで選択解除。

IV

データモデル / Data Model

グループは再帰的な木構造（親グループを持てる）。ワードは必ずいずれかのグループに属する。

// Group：任意の深さでネスト可能
{
  "id": "grp_character",
  "name": "CHARACTER",
  "collapsed": false,
  "groups": [ /* 子グループ配列 */ ],
  "words": [
    {
      "id": "w_001",
      "text": "long silver hair",
      "note": "銀髪ロング。綾波系。",
      "selected": false
    }
  ]
}
// ルート構造
{ "version": 1, "rootGroups": [ /* Group[] */ ] }
同一ワードの重複は text の正規化（trim・小文字化）で判定し、最終プロンプト出力時に排除。語順は選択された順ではなく、左ツリー上の出現順を維持。
V

グループ機能 / Groups

階層と命名

グループ名・ワード共に自由入力。グループは無制限にネスト可能（例：CHARACTER > Upper Body > Hair）。

折り畳み

ワード増加による縦長化を防ぐ折り畳み機能。グループ内に選択ワードがある場合、グループ行に蛍光緑の徽章（badge）と発光で「内に選択あり」を暗示する。

操作対象動作
グループ名 / シングルクリック階層の展開・折り畳み切替
グループ名 / ダブルクリックグループ名編集モードへ移行
VI

ワード機能 / Words

選択と編集

シングルクリック＝選択切替。ダブルクリック＝ワード編集。

注釈

各ワードに注釈（note）を付与。意味不明なワードも即座に理解可能。ワード横に小さな緑の印で注釈の有無を示す。

検索

ワード本文および注釈を横断検索する検索欄。ヒット以外を淡色化し、コンテキストを保つ。

VII

総括欄 / Synthesis

最終プロンプト生成

選択された全ワードを、左ツリーの出現順に集約。text の正規化比較で重複を排除し、カンマ（または改行）区切りで右上の総括欄へ出力。

  • 出力形式：カンマ区切り一行（コピー容易）。切替で改行区切りも可。
  • 右下「選択済み」欄：各ワードをクリックすると選択解除され、総括欄が即時再計算。
  • クリップボードコピー（緑のアイコン）を備える。
VIII

ドラッグ&ドロップ / Drag & Drop

読みやすさのため、後からの追加にも耐えるよう並び替えを前提とする。スムーズなアニメーション（Motion spring）を伴う。

対象移動範囲備考
ワード同一グループ内のみ順序の並び替え。他グループへは移動不可。
グループ順序の並び替え ＋ 別グループ内への移動グループ A をグループ B の子としてネスト可能。

アニメーション要件

  • ドラッグ開始：対象が縮小＋半透過、緑の残像Trailを引く。
  • ドロップ先：spring による弾性のある挿入（LayoutGroup + AnimatePresence）。
  • 挿入位置インジケータ：緑の発光ラインで挿入点を表示。
  • ドロップ終了：ねばりのある spring で着地（stiffness≈260, damping≈24 程度）。

実装方針

HTML5 DnD ではなく、Motion の Reorder.Group / Reorder.Item と layout prop を活用。グループ間移動は状態ツリーの再帰書き換えで実現し、layoutId で連続性を保持。

IX

JSON 入出力 / Import & Export

アイコンのみの操作

文字を排した2つのアイコンのみで完結。

  • 書き出し（Export）：赤紫の下向き矢印アイコン → 現在のツリーを JSON に固めてダウンロード。
  • 読み込み（Import）：緑の上向き矢印アイコン → ファイル選択ダイアログから JSON を読み込み、ツリーを置換（マージ確認付き）。
  • アイコンは React Icons（Fi/Fa）から選定、ホバー時に発光。
X

技術スタック / Stack

React 18 Vite TypeScript Tailwind CSS Motion React Icons

状態管理

React Context（PromptContext）でグローバル状態を一元管理。ツリー構造・選択状態・折り畳み状態を保持し、Provider 経由で各画面へ供給。

永続化層は Context 内に抽象化し chrome.storage.local へ自動保存（debounce）。

ビルド出力

Vite で popup.html を起点にビルド。出力物 + manifest.json を dist/ にまとめ、Chrome に「パッケージ化されていない拡張機能」として読み込ませる。

manifest_version: 3 / action.default_popup: popup.html
